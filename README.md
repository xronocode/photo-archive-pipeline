# Псевдокод системы: дедуп, тегирование и хронологический разбор фотоархива (AI-агент + локальные модели)

Это **универсальное** описание системы, которая приводит в порядок любой большой разрозненный
медиаархив: убирает дубли, понимает, что на снимках, размечает людей и раскладывает всё в
хронологическую структуру `ГГГГ / ГГГГ-ММ-ДД — событие` с тегами людей внутри файлов.
Источники могут быть любыми — облако с папками по годам, свалки «разобрать», почта, телефон.

Все конкретные числа ниже — **пороги-параметры** (их калибруют под свой архив) и **примеры
из одного реального прогона** (их приводят для масштаба, не как требование). Код универсален;
цифры — иллюстрация.

> Пример прогона, к которому привязаны примеры ниже: ~36 000 фото/видео, 222 ГБ, копившиеся
> 20 лет, размазанные по облаку (годы + свалки «разобрать»), почте и Apple Photos.

Описание самодостаточно: архитектура и псевдокод каждого шага — что гоняется, какими
моделями, с какими порогами и где грабли. Язык намеренно не указан: это одноразовые скрипты,
которые агент пишет под конкретный шаг. Псевдокод — про логику, а не про синтаксис. Чтобы
воспроизвести, ничего, кроме этого файла, не нужно.

> Главный принцип всей системы: **тяжёлый перемол — локальным моделям (почти бесплатно),
> оркестрация и контроль качества — фронтир-агенту, финальные решения — человеку.**
> И второе правило: **всё обратимо** — удаляем в Корзину/карантин, каждое перемещение в undo-логе.

## Окружение (пример: на чём гонялось у меня)

Всё крутилось локально, без облака и без GPU-сервера. Это не требование системы, а пример
конфигурации — она прекрасно живёт и на других Apple Silicon, и на ПК с GPU:

- **MacBook Air, Apple M5, 10 ядер (4 perf + 6 eff), 32 ГБ unified memory**, macOS 26.
- CLIP — на MPS (встроенный GPU); VLM (Qwen2.5-VL-7B, 4-bit) — нативно через **MLX**
  (а не Ollama — на Apple Silicon заметно быстрее); InsightFace (лица) — на CPU.
- Системный Python 3.9; ffmpeg/exiftool из brew.

Почему «за вечер» — это реально: кластеризация лиц сама по себе быстрая (это эмбеддинги +
kNN, не VLM). Дорогой VLM гнался **только по выборке** (пара кадров на кластер, а не весь
архив), а детект лиц по всему архиву — это пара часов фоновой молотьбы на CPU, во время
которой можно просто заниматься своими делами. Итого: пара часов активной работы + «оставил
молоть».

        \             |                  |                 /
         └────────────┴──────┬───────────┴────────────────┘
                             ▼
                    ФРОНТИР-АГЕНТ (оркестратор)
                    - пишет одноразовый скрипт под шаг
                    - дёргает инструменты
                    - САМ проверяет результат (есть зрение)
                             │
            раздаёт тяжёлую работу ▼
        ┌────────────┬───────────────┬──────────────┬────────────┐
     czkawka      open_clip      Qwen2.5-VL      InsightFace   exiftool
   (дедуп по    (грубые теги    (имена событий  (лица →       (теги людей
    хэшу обр.)   содержимого)    по кадрам)      кластеры)     внутрь файлов)
        └────────────┴───────────────┴──────────────┴────────────┘
                             │
                             ▼

Стек: оркестратор — Claude Code; дедуп — czkawka; грубые теги — open_clip (CLIP ViT-B-32);
описания событий — Qwen2.5-VL-7B через mlx-vlm (Apple Silicon); лица — InsightFace
(ArcFace); запись тегов — exiftool. Всё локально, без облака и без внешней галереи.

## Шаг 0. Почта (с чего начали)

```
connect Gmail API (scope = modify: читать + Корзина, НЕ permanent delete)

analyze:
    выгрузить метаданные всех писем (батч-запросами с retry+backoff)
    # ГРАБЛИ: батч молча терял ~21k писем → ловить ШИРОКИЙ Exception,
    #         не только HttpError, и повторять с экспон. задержкой
    сгруппировать по отправителю / размеру / году → раскладка

cleanup spam:
    кандидаты = явная реклама/уведомления
    → переместить в Корзину (НЕ стирать)
    # урок: в спам уехала рассылка живого интернет-магазина (рабочая история).
    #        спасло то, что только Корзина → вернули одной командой, отправителя в исключения

extract photo-archives from heavy mail:
    тяжёлые письма ≈ непросмотренный фотоальбом, событие размазано по письмам "часть 1/2/3"
    агент читает темы+содержание, СМОТРИТ сами вложения
    → группирует не "папка на письмо", а "папка на реальное событие"
    → импорт в Apple Photos
```

## Шаг 1. Дедуп — до всего остального

Сравнивать не байты, а изображения (перцептивный хэш). Точные хэши не ловят
пересохранения/ресайзы/кадры серий.

```
for каждая осмысленная папка архива:          # дедуп ВНУТРИ папки, не вперемешку
    groups = czkawka_image(folder,
                           hash_size = 16,
                           max_difference = 5)   # строгий порог, откалиброван на реальной папке
    for g in groups:
        keep   = argmax(g, by = (resolution, file_size))   # лучший по разрешению/размеру
        rest   = g \ {keep}
        for f in rest:
            move(f → quarantine/<относительный путь>)        # ОБРАТИМО, не стираем

# Браузерный ревьюер (Flask, localhost): группы, миниатюры в ряд, лучший предвыбран "оставить".
# Но по факту выбрали авто-режим: "схема работает идеально" → гнать без ручного ревью.
```

**Грабли czkawka_cli image** (реальные, стоили времени):
- `-s / --max-difference` — это порог. **Не** `-m` (это `minimal-file-size`).
- `-C / --compact-file-to-save` — JSON. **Не** `-c` (это `hash-size`!).
- APFS регистронезависим: `dedup_Liz.json` и `dedup_liz.json` — один файл.

## Шаг 2. «Что на фото?» — локальные модели

### 2a. Грубые теги содержимого (CLIP)

```
load CLIP (open_clip, ViT-B-32) на MPS (GPU Apple Silicon)
vocab = ["пляж","дети","еда","документ","скриншот","праздник", ...]

for photo in all_photos:
    emb_img  = clip.encode_image(photo)
    tags     = top_k( cosine(emb_img, clip.encode_text(vocab)) )
    date,gps = exif(photo)
    write {photo, tags, date, gps} → tags.jsonl
    # польза: отделить документы/скриншоты от настоящих снимков
```

### 2b. Имена событий (VLM)

Тег «праздник» — не имя альбома. Нужно «день рождения», «поездка», «корпоратив».

```
load Qwen2.5-VL-7B-4bit через mlx-vlm

for cluster in clusters:                   # кластер = подпапка свалки (или год-месяц внутри неё)
    sample = 2 репрезентативных кадра
    sample = downscale(sample → 1024 px)   # ГРАБЛИ: иначе Qwen2.5-VL пытается выделить 84 ГБ Metal
    caption = vlm(sample, "опиши одной фразой: событие, кто, где")
    write {cluster, caption} → captions.jsonl
    # выдаёт "Дети на катке с маскотом-зайцем" → из этого собирается имя события

# ЭКОНОМИКА: ~17 c/кадр на 7B. Гнать ТОЛЬКО выборку (пара кадров на кластер), НЕ весь архив.
# Просить фронтир-модель "опиши все фото" — долго и дорого по токенам; VLM локально — почти даром.
```

**Грабли VLM:**
- `transformers` откатить до **4.49.0** (свежий fast image-processor несовместим с mlx-vlm 0.1.15).
- Скрипт делать **возобновляемым** (пропускать уже сделанные ключи) — прогон длинный.
- **НЕ** запускать VLM (GPU) одновременно с InsightFace (CPU, тяжёлый) → OOM-kill (у меня на 32 ГБ).

## Шаг 3. Лица — как диаризация спикеров

Задача «кто на фото» = диаризация в аудио: разбить на «персона 1/2/...», человек называет
каждую один раз, имя расходится по всем снимкам.

```
# (1) детект + эмбеддинги
load InsightFace buffalo_l (ArcFace), CPU/onnxruntime
for photo in all_photos:
    for face in detect(photo):
        if face.det_score < 0.62: continue
        append(emb.npy, face.normed_embedding)   # 512-d, L2-норм.
        append(meta.jsonl, {photo, bbox})
# пример прогона для масштаба: ~74 000 лиц на ~26 000 фото

# (2) кластеризация — kNN + union-find (не наивный N²)
for chunk in emb:                          # чанками, чтобы влезло в память
    knn = topk(chunk @ emb.T, k = 15)      # косинус = скалярное (вектора норм.)
    for (i, j, sim) in knn:
        if sim >= 0.72:                    # КЛЮЧЕВОЙ ПОРОГ
            union(i, j)
clusters = connected_components(union_find)
# пример прогона: ~2153 чистые персоны; топ-персоны по 3-4 тыс. фото каждая
```

**Грабли порога (поучительный провал):** при 0.5–0.65 кластеризация схлопывала пол-архива в
один мегакластер (в моём прогоне — десятки тысяч лиц, мужчины+женщины вперемешку) — эффект
цепочки через «мостиковые» лица. **0.72** дал чистые кластеры. Порог — главное, что калибруют.

**Кто проверял качество:** не человек глазами, а сам агент — вырезал по несколько лиц из
крупнейших кластеров и смотрел (у фронтир-модели есть зрение), один это человек или каша.

```
# (3) человек называет — Flask-лейблер (localhost)
персоны отсортированы по частоте (часто в кадре сверху)
человек подписывает ТОЛЬКО верхушку → names.json   # в примере ~27 персон из 2153
# отдача максимальна у частых; хвост случайных людей не нужен
```

## Шаг 4. Видео — тот же стек, в два прохода

Отдельный видео-стек не нужен: ffmpeg достаёт кадры → дальше те же модели.

```
# что в ролике
for video in videos:
    frames = ffmpeg_extract(video, n = 3, scale = 768px, at = [10%, 50%, 90%])
    caption = vlm(frames)                  # та же Qwen2.5-VL
    write {video, caption}

# кто в ролике — матч к УЖЕ названным персонам
bank = эталоны: эмбеддинги из именованных кластеров (≤40 на персону) + их имена

pass1: for video in all_videos:
           frames = ffmpeg_extract(video, n = 5)
           found  = {}
           for f in frames: for face in detect(f):
               j = argmax(bank @ face.emb)
               if sim[j] >= 0.55: found += bank_name[j]
           write {video, people: found}

pass2: for video where pass1 нашёл НИКОГО:    # 5 кадров пропускают мелькнувшего человека
           прогнать плотнее, n = 12 кадров

# пример прогона: 1754 ролика → люди в 470 (проход 1) + ещё 138 (проход 2) = 608

# теги людей → в видеофайл
exiftool(video, XMP-dc:Subject += name, Keywords += name,
         api QuickTimeUTC=1, codedcharacterset=utf8)
```

Логика та же, что везде: **сначала дёшево закрыть основной объём, потом точечно дожать остаток.**

**Грабли:** exiftool не пишет теги в `.avi/.vob` (RIFF) — детект всё равно сохраняем в
каталог `video_people.jsonl`. Для `mp4/mov` нужен `-api QuickTimeUTC=1`.

## Шаг 5. Сортировка — и почему EXIF врёт

Целевая структура: `ГГГГ / ГГГГ-ММ-ДД — событие`. Через манифест с предпросмотром и полным
undo-логом, сперва пилот на одном годе.

```
build_manifest (dry-run, ничего не двигает):
    for file in all_media:
        event = из имени уже-разложенной папки  ИЛИ  из правил для свалок (+VLM-имя)

        # ДАТА — ключевое правило приоритета:
        exif = exif_date(file) if valid_year(exif) else None
        fnd  = date_from_filename(file)
        fold = date_from_folder_name(file)        # "2008-09 корпоратив" → 2008-09
        if fold:
            if   exif and year(exif) == year(fold): date = exif   # папка и EXIF согласны → точная дата EXIF
            elif fnd  and year(fnd)  == year(fold): date = fnd
            else:                                   date = fold    # РАСХОЖДЕНИЕ ГОДА → верим папке
        else:
            date = exif or fnd
        clamp year ∈ [1990, 2026]

        dst = "{year}/{date} — {event}/{filename}"
        people = lookup(file)                     # из реестра лиц
        append manifest {src, dst, date, event, people}

apply (do_sort):
    for m in manifest:
        move(m.src → m.dst);  log undo {from: dst, to: src}      # ОБРАТИМО
        for name in m.people:
            exiftool(m.dst, XMP-dc:Subject += name, Keywords += name)   # += не затирая, UTF-8
```

**Почему верим папке, а не EXIF (пилот 2008→2005 это поймал):**
- у старых камер с несбитыми часами EXIF по умолчанию `2005-01-01` / `2000`;
- у сканов EXIF = дата **оцифровки**, не съёмки;
- человек, который называл папку, знал год точно — камера нет.

## Шаг 6. Гранулярность (день → месяц/год)

Первый авто-результат дробил одно событие на папки по дням. Чиним в два прохода.

```
# проход 1: схлопнуть дни в МЕСЯЦ для разовых событий
group folders by (year, event)
for каждой (year, event):
    if все папки события в одном месяце → "ГГГГ-ММ — событие"
    собрать дни-папки в одну месячную

# проход 2: "папки-люди" и длящиеся истории → в ГОД
for event:
    if event занимает >= 3 месяцев  ИЛИ  имя похоже на человека/субъект
       (Лиза/Миша/Карина/ремонт/dacha/...):
        → "ГГГГ — субъект"        # это субъект, а не разовое событие
    else:
        оставить помесячно         # поездки, праздники, конференция, свадьба

# оба прохода → в undo-лог. Пример из прогона: один год ~200 папок-по-дням → 49.
```

## Инварианты (то, что держит систему безопасной)

1. **Удаляем только обратимо:** почта → Корзина, дубли → карантин, перемещения → undo-лог.
   Никаких permanent delete, пока человек не подтвердил.
2. **Дедуп — раньше всего:** иначе вся дальнейшая работа множится на копиях.
3. **Дёшево-сначала, дорого-точечно:** основной объём — дешёвым проходом, остаток — плотным.
4. **Имя папки > EXIF при расхождении года.**
5. **Никогда не гонять VLM(GPU) и InsightFace(CPU) одновременно** на машине с ограниченной памятью.
6. **Все долгие прогоны возобновляемые** (пропуск уже сделанного по ключу).
7. **Контроль качества — на агенте, где можно** (у фронтир-модели есть зрение), человек — на
   решениях, которые модель знать не может.

## Как воспроизвести

Достаточно скормить **этот файл** любому толковому AI-агенту-оркестратору: дальше он сам
напишет одноразовые скрипты под ваш архив и дёрнет нужные инструменты. Конкретные числа
(пороги 0.72 / max-difference 5, размер выборки для VLM) калибруются на вашей реальной папке —
именно так они и подбирались здесь.
